# Cellpose Training Support — Design Spec

**Date:** 2026-03-30
**Scope:** Container changes to W_Segmentation-Cellpose4 to support training via the biomero `run_workflow()` pipeline.

## Context

The biomero library and OMERO.biomero have been updated to handle training workflows. The Cellpose container (`W_Segmentation-Cellpose4`) currently only supports inference. This spec covers adding a training entry point to the same container, so a single image supports both inference and training — selected by how it's invoked.

Training jobs flow through the same `run_workflow()` → `get_workflow_command()` → `job_template.sh` → `singularity run` pipeline as inference. The container receives `--infolder`, `--outfolder`, `--gtfolder`, env vars from workflow params, and `$MODELS_PATH` bound to `/tmp/models`.

## Data Structure

The OMERO script (separate scope) prepares the following directory layout before transferring to Slurm:

```
data/
├── config.yaml
├── in/
│   ├── train/           # training images (TIFF)
│   ├── validation/      # validation images (TIFF)
│   └── test/            # held-out test images (TIFF)
└── gt/
    ├── train/           # training masks (instance label TIFF, matching image filenames)
    ├── validation/      # validation masks
    └── test/            # test masks
```

### config.yaml

Generated by the OMERO script. Primary source of training parameters; env vars serve as fallback/override.

```yaml
training:
  pretrained_model: "cpsam"       # base model to fine-tune
  n_epochs: 100
  learning_rate: 0.00001
  weight_decay: 0.1
  batch_size: 1
  diameter: 30
  model_name: "my_custom_model"   # user-friendly display name
  channels: [0, 0]                # [cytoplasm_channel, nucleus_channel]

metadata:
  source_datasets: [42, 87]
  trained_by: "user5"
  workflow_name: "W_Segmentation-Cellpose4"
```

## Container Changes

### 1. `entrypoint.sh` (new)

Dispatcher script that replaces the current inline `ENTRYPOINT`:

```bash
#!/bin/bash
source /opt/conda/etc/profile.d/conda.sh
if [ "$TRAINING_MODE" = "true" ]; then
    conda activate cellpose_env
    exec python /app/train.py "$@"
else
    conda activate cytomine_py37
    exec python /app/run.py "$@"
fi
```

- `TRAINING_MODE` env var is set by biomero via `workflow_params_to_envvars()` when the OMERO script passes `training_mode=true` as a workflow kwarg.
- Inference path (`run.py`) is completely unchanged.

### 2. `train.py` (new)

Training entry point. Responsibilities:

**a) Parse arguments and config**
- Accepts `--infolder`, `--outfolder`, `--gtfolder` CLI args (same as inference, where `--infolder` = `data/in`, `--gtfolder` = `data/gt`, `--outfolder` = `data/out`)
- Reads `<infolder>/../config.yaml` (resolves to `data/config.yaml` since `--infolder` is `data/in`)
- Falls back to env vars for any missing parameters
- If `config.yaml` is not found, uses env vars only (backwards compatible)

**b) Prepare cellpose training directories**

Cellpose expects images and corresponding `_masks.tif` files in the same directory. `train.py` creates temporary directories and symlinks/copies:

```
/tmp/cellpose_train/
├── image1.tif
├── image1_masks.tif
├── image2.tif
├── image2_masks.tif
...

/tmp/cellpose_val/
├── image3.tif
├── image3_masks.tif
...
```

Images come from `--infolder/train/` and `--infolder/validation/`. Masks from `--gtfolder/train/` and `--gtfolder/validation/`, renamed to `<image_stem>_masks.tif`.

**c) Run cellpose training**

Calls the cellpose CLI via subprocess:

```bash
cellpose --train \
    --dir /tmp/cellpose_train \
    --test_dir /tmp/cellpose_val \
    --pretrained_model <pretrained_model> \
    --n_epochs <n_epochs> \
    --learning_rate <learning_rate> \
    --weight_decay <weight_decay> \
    --train_batch_size <batch_size> \
    --diameter <diameter> \
    --verbose
```

With `--use_gpu` if available.

**d) Model naming**

The model ID (filesystem name) is generated as:

- If `model_name` provided in config: `<model_name>_<YYYYMMDD_HHMMSS>`
- Fallback: `<workflow_name>_<user_id>_<YYYYMMDD_HHMMSS>`

Example: `my_custom_model_20260330_143022`

**e) Save model**

- Copy trained model files to `/tmp/models/<model_id>/` (persists on Slurm via `$MODELS_PATH` bind)
- Zip the model directory and place in `--outfolder/<model_id>.zip`

**f) Write training results**

Write `training_results.yaml` to `--outfolder`:

```yaml
model_id: "my_custom_model_20260330_143022"
model_name: "my_custom_model"
source_datasets: [42, 87]
trained_by: "user5"
pretrained_model: "cpsam"
n_epochs: 100
learning_rate: 0.00001
final_loss: 0.023
validation_metrics:
  avg_precision: 0.85
```

**g) Optional: evaluate on test set**

If `--infolder/test/` and `--gtfolder/test/` exist and are non-empty, run inference on the test set and compute metrics. Write results to `training_results.yaml` under a `test_metrics` key.

### 3. Dockerfile changes

```dockerfile
# Add new files
COPY train.py /app/train.py
COPY entrypoint.sh /app/entrypoint.sh
RUN chmod +x /app/entrypoint.sh

# Add PyYAML to cellpose_env for config parsing
RUN conda run -n cellpose_env pip install --no-cache-dir pyyaml

# Update entrypoint
ENTRYPOINT ["/app/entrypoint.sh"]
```

### 4. `descriptor.json`

No changes. Training parameters come via `config.yaml`, not the descriptor. The descriptor remains inference-focused.

## Model Output & Provenance

After training completes and biomero retrieves `--outfolder`:

| Artifact | Location | Purpose |
|----------|----------|---------|
| Model files | `$MODELS_PATH/<model_id>/` on Slurm | Available for inference via custom model selector |
| Model zip | OMERO FileAnnotation (ns: `biomero.training.model`) | Provenance, sharing, download |
| Training results | OMERO map annotation from `training_results.yaml` | Metrics, hyperparams, lineage |

## Out of Scope

These are needed for the full training pipeline but are separate work items:

- **`SLURM_Run_Training.py`** — new OMERO script that reads annotate_ai config, fetches images, rasterizes ROIs to masks, organizes train/validation/test splits, generates `config.yaml`, and calls `run_workflow()` with `training_mode=true`
- **OMERO.biomero UI** — training workflow trigger from the web interface
- **Model management UI** — listing, deleting, renaming trained models
- **biomero library changes** — handling model FileAnnotation upload from `--outfolder` results

## Cellpose Training Reference

Cellpose 4.x training CLI:

```bash
cellpose --train --dir <train_dir> --test_dir <test_dir> \
    --pretrained_model <model> --n_epochs 100 \
    --learning_rate 1e-5 --weight_decay 0.1 \
    --train_batch_size 1 --diameter 30 --use_gpu
```

- Expects images + `_masks.tif` files in the same directory
- Models saved to `$CELLPOSE_LOCAL_MODELS_PATH` (set to `/tmp/models/cellpose/` in the container)
- Recommended fine-tuning settings for cpsam: lr=1e-5, weight_decay=0.1, batch_size=1
